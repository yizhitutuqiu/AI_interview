# AI面试工具产品需求与架构文档 (PRD & Architecture)

## 1. 产品概述
本项目旨在开发一个基于Web的AI面试工具。用户可以通过网页与AI面试官进行交互，支持文本输入和语音转文字输入（Speech-to-Text）。AI面试官由豆包大模型（Volcengine）提供能力支持，针对用户的回答进行追问、点评和打分。

## 2. 核心功能需求
- **对话界面**：类似聊天软件的界面，展示AI面试官与用户的对话流。
- **输入方式**：
  - **文本输入**：标准的文本框和发送按钮。
  - **语音输入**：支持点击麦克风按钮后进行语音说话，前端自动将其转换为文字并填入输入框（基于浏览器的Web Speech API）。
- **AI面试官（豆包API）**：
  - 接收用户的消息并生成专业的面试回复。
  - 维护对话上下文，支持多轮连贯的面试问答。

## 3. 技术架构
本项目采用前后端分离的极简架构：

### 3.1 前端 (Frontend)
- **技术栈**：纯 HTML + CSS + 原生 JavaScript。
- **目录**：`/html`
- **核心逻辑**：
  - 负责界面的渲染和交互（消息气泡流）。
  - 调用浏览器原生的 `webkitSpeechRecognition` API 实现语音转文字。
  - 通过 `fetch` 将用户消息发送给后端 API，并处理后端返回的 AI 回复。

### 3.2 后端 (Backend)
- **技术栈**：Python + FastAPI。
- **核心逻辑**：
  - 静态文件服务：将前端 `/html` 目录下的静态文件托管出去。
  - `/api/chat` 接口：接收前端传来的对话历史，组装后调用火山引擎（豆包）API。
  - 宏定义：代码中保留 `YOUR_DOUBAO_API_KEY` 和 `YOUR_ENDPOINT_ID` 作为宏，供开发者自行填写。

## 4. 接口设计
- **请求路径**：`POST /api/chat`
- **请求体 (JSON)**：
  ```json
  {
    "messages": [
      {"role": "user", "content": "你好，我来面试"},
      {"role": "assistant", "content": "你好，请先做一个简单的自我介绍。"}
    ]
  }
  ```
- **响应体 (JSON)**：
  ```json
  {
    "reply": "好的，根据你的经历，你在上一份工作中遇到过最大的挑战是什么？"
  }
  ```
